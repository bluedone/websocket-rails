    Websocket-Rails

    Now with streaming HTTP support for improved browser compatibility

    We make WebSockets too easy not to use. Websocket-Rails integrates a Rack compatible WebSocket server directly into your Rails application without the need for a separate server process. Client side events are mapped to controller actions using the event router. It's the power of the evented web with the development flow you are used to.

    Check out the documentation for detailed information

    As this project is in its infancy, all suggestions, gripes, pull requests, and general nay saying is very much appreciated. Please feel free to open up an issue on the Github issue tracker, or shoot us an email.

    5 Easy Steps to WebSocket Goodness

    1. Add the websocket-rails gem to your Gemfile and install the bundle.
    2. Create a WebsocketRails Controller in your app/controllers directory.
    3. Map a client event to your controller action using the Event Router.
    4. Launch the web server with the included executable. (This is temporary)
    5. Connect to the server with a WebSocket client and enjoy.

    Project Goals

    Websocket-Rails aims to be more than just a WebSocket server. We wish to provide an alternative means for easily developing rich interactive web applications without pushing application logic onto the client side and fragmenting the code base. Client code should be responsible for presentation. Server code should encapsulate the domain logic. Life should remain enjoyable.

    This project will eventually include a full featured JavaScript WebSocket client and/or integration into something like EmberJS to make the client event management as streamlined as possible. Ideally, it will grow to a point where important client events naturally bubble up to the server for processing and any time a model is updated in Rails, events will trigger the JavaScript client to automatically update the rendered representation of that model on the client with minimal effort.

    Current Development Needs

    1. JavaScript WebSocket based event dispatcher.
    2. Logger integration for easy event logging.
    3. ActiveModel event integration.
    4. Event namespace support with scoping handled in the event route file.
